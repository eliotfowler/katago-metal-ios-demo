# KataGo iOS Metal Demo

An open-source implementation of [KataGo](https://github.com/lightvector/KataGo) running on iOS with Metal GPU acceleration. Uses [ChinChangYang's Metal+CoreML fork](https://github.com/ChinChangYang/KataGo/tree/metal-coreml-stable) compiled directly inside an Xcode project — no CMake required.

---

## What this is

A working iOS app that runs KataGo's full MCTS analysis engine on-device, GPU-accelerated via Apple's Metal Performance Shaders Graph (MPSGraph) framework. The app lets you select a neural net size (b6 / b10 / b15 / b18), a board size, and a visit budget, then runs analysis on a fixed test position and displays move evaluations in real time.

This project exists as a reference for iOS developers who want to integrate KataGo into their own apps.

## Why it exists

KataGo's Metal backend uses Swift/C++ bidirectional interop, which CMake cannot cross-compile for iOS — even with a custom toolchain, the Swift compiler targets macOS rather than the iOS device SDK. The only reliable path is building natively inside Xcode, which handles Swift/C++ interop correctly. See [`docs/metal-backend-investigation.md`](docs/metal-backend-investigation.md) for the full investigation.

## Requirements

- **Xcode 16+** with the iOS 17+ SDK
- **Physical iOS device or Apple Silicon-based mac** — Metal does not work in the simulator for intel-based macs.
- **ARM64 device** (iPhone 12 or later recommended for performance)
- **Neural net weight files** (`.bin.gz`) — downloaded by `scripts/setup.sh`

## Quick start

```bash
# 1. Clone with submodules
git clone --recurse-submodules https://github.com/eliotfowler/KataGoMetalDemo.git
cd KataGoMetalDemo

# 2. Set up everything (submodules, source copy, weight download)
./scripts/setup.sh

# 3. Open in Xcode and build on device
open KataGoMetalDemo.xcodeproj
```

Select your physical device as the build target and press **⌘R**. The first build takes several minutes (compiling ~70 KataGo C++ files and the Swift Metal backend). The first run on a device also takes 10–20 seconds to compile Metal shaders — this is cached for subsequent runs.

## How it works

```
SwiftUI (ContentView)
    └── KataGoService.swift       (Swift — @MainActor ObservableObject)
            └── katago_bridge.h/.cpp   (C bridge — thin wrapper over KataGo C++ API)
                    └── KataGo C++ engine (MCTS search, neural net eval)
                            └── Metal GPU (via MPSGraph — Apple Metal backend)
```

### Architecture details

1. **Two Xcode targets** — `KataGoEngine` (static library) and `KataGoMetalDemo` (the app). Keeping the engine in a separate target isolates its C++17 / Swift interop settings from the app target's defaults.

2. **KataGoEngine/ directory** — Generated by `scripts/copy_katago_files.sh`. Contains the subset of KataGo C++ source files needed for on-device inference. Uses Xcode's folder reference (file system synchronization) so new files are picked up automatically on re-copy.

3. **C bridge** — Swift cannot directly import KataGo's C++ headers (even with Swift/C++ interop, the KataGo headers use features not yet supported). `katago_bridge.h` exposes a minimal C API. All structs use fixed-size arrays so they cross the C ABI boundary safely without heap allocation.

4. **`USE_COREML_BACKEND=1` preprocessor macro** — Despite the name, this enables the Metal + CoreML combined backend in KataGo, which uses MPSGraph for the primary GPU computation. CoreML is an optional accelerator within that backend.

5. **Off-thread initialization** — `katago_create` reads weights and compiles Metal shaders. `KataGoService` runs this on a `Task.detached` so the main actor is never blocked.

## Performance

Tested on **iPhone 15 Pro**, 19×19 board:

| Model      | Params    | Size    | 100 visits | 500 visits | 1000 visits |
| ---------- | --------- | ------- | ---------- | ---------- | ----------- |
| b6c96      | 6 blocks  | ~3.6 MB | ~0.5s      | ~1.8s      | ~2.1s       |
| b10c128    | 10 blocks | ~11 MB  | ~0.8s      | ~2.1s      | ~2.4s       |
| b15c192    | 15 blocks | ~35 MB  | ~1.2s      | ~3s        | ~3.5s       |
| b18c384nbt | 18 blocks | ~120 MB | ~2.1s      | ~7.1s      | ~8.8s       |

For comparison, b15 on the same device using the Eigen (CPU) backend takes ~20s at 1000 visits.

### Neural net selection

| Model      | Use case                                     |
| ---------- | -------------------------------------------- |
| b6c96      | Fast feedback, real-time apps, weak hardware |
| b10c128    | Good balance of speed and strength           |
| b15c192    | Strong analysis, ~3–5s per query             |
| b18c384nbt | Strongest available, best for deep analysis  |

## Project structure

```
KataGoMetalDemo/
├── katago/                    # Git submodule — ChinChangYang/KataGo (metal-coreml-stable branch)
├── KataGoEngine/              # Generated by scripts/copy_katago_files.sh — DO NOT EDIT DIRECTLY
│   ├── core/, game/, search/, neuralnet/, program/
│   └── (Swift Metal backend files)
├── App/                           # SwiftUI app source (file system synchronized folder)
│   ├── ContentView.swift
│   ├── KataGoService.swift        # Main Swift/C++ integration layer
│   ├── KataGoMetalDemoApp.swift
│   └── TestPositions.swift
├── KataGoMetalDemo/
│   ├── Bridge/
│   │   ├── katago_bridge.h        # C API exposed to Swift
│   │   ├── katago_bridge.cpp      # C++ implementation
│   │   └── KataGoMetalDemo-Bridging-Header.h
│   ├── Resources/
│   │   ├── analysis.cfg           # KataGo search configuration
│   │   └── model_*.bin.gz         # Neural net weights (gitignored)
│   └── Info.plist
├── KataGoMetalDemo.xcodeproj/     # Main Xcode project
├── docs/
│   └── metal-backend-investigation.md   # Historical: why CMake doesn't work
└── scripts/
    ├── setup.sh                  # One-shot setup (submodules + copy + download)
    ├── copy_katago_files.sh      # Copies sources from katago/ → KataGoEngine/
    └── download_weights.sh       # Downloads neural net weights
```

### Build settings reference

**KataGoEngine target:**

| Setting                    | Value                                                                                                           |
| -------------------------- | --------------------------------------------------------------------------------------------------------------- |
| Preprocessor Macros        | `USE_COREML_BACKEND=1`, `NO_LIBZIP`, `NO_GIT_REVISION`, `OS_IS_IOS`, `COMPILE_MAX_BOARD_LEN=19`                 |
| C++ Language Dialect       | GNU++17                                                                                                         |
| Swift Objc Interop Mode    | `objcxx`                                                                                                        |
| Product Module Name        | `KataGoSwift`                                                                                                   |
| Header Search Paths        | `$(SRCROOT)/katago/cpp`, `$(SRCROOT)/katago/cpp/external`, `$(SRCROOT)/katago/cpp/external/tclap-1.2.2/include` |
| System Header Search Paths | `$(SRCROOT)/katago/cpp/external/filesystem-1.5.8/include`                                                       |
| Linked Frameworks          | Metal, MetalPerformanceShaders, MetalPerformanceShadersGraph, CoreML, Accelerate                                |

**KataGoMetalDemo (app) target:**

| Setting             | Value                                                                                                 |
| ------------------- | ----------------------------------------------------------------------------------------------------- |
| Preprocessor Macros | `USE_COREML_BACKEND=1`, `NO_GIT_REVISION=1`, `NO_LIBZIP=1`, `COMPILE_MAX_BOARD_LEN=19`                |
| Linked Libraries    | `libKataGoEngine.a`, Metal, MetalPerformanceShaders, MetalPerformanceShadersGraph, CoreML, Accelerate |
